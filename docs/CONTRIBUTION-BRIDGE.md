# Contribution Bridge

## Why GitHub Matters Here

Inside `K Mentorship Hub`, GitHub is not just storage.

It is the place where learning becomes visible.

That means you do not need to arrive as an expert to contribute.
You can start by making the shared surface:

- clearer
- easier to navigate
- easier to run
- easier to understand
- easier to extend

## Best First Contribution Types

Start small.

The strongest early contributions are often:

- docs improvements
- reproducibility notes
- glossary pages
- small data notes
- visual summaries
- dashboard mockups
- starter checklists
- `academy-practice` tasks

## Suggested Labels

- `good first issue`
- `academy-practice`
- `docs`
- `reproducibility`
- `data`
- `visualization`
- `research-review-needed`
- `safe-for-fork`
- `SET-S`
- `SET-E`
- `SET-T`

## How To Read The Labels

- `SET-S` -> science-facing work
- `SET-E` -> product, venture, market, or ops-facing work
- `SET-T` -> code, systems, data, tooling, or infrastructure-facing work

## Fork Or PR?

Use a `fork` when:

- you are experimenting
- you want to learn by trying
- the output is still rough
- you are not sure the work belongs in the shared repo yet

Open a `PR` when:

- you improved clarity
- you fixed a broken step
- you added something reusable
- your output would help the next person

## The Best Contribution Rhythm

1. pick one small issue
2. understand the current surface
3. improve one thing
4. explain what changed
5. share it cleanly

That is more valuable than one oversized contribution that nobody can review.
