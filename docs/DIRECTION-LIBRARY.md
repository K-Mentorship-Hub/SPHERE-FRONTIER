# Direction Library

## What This Page Is For

This page is the readable map behind the dashboard.

Use it when you want to:

- understand where a direction belongs inside `S / E / T`
- choose one practical starting point
- find starter resources without opening twenty tabs
- translate an interest into a visible output
- understand when to `fork` and when to open a `PR`

## The Fastest Rule

Start with one direction, not your whole future.

The best first move is:

1. pick one direction
2. open one starter resource
3. complete one practice task
4. publish one visible output
5. open a `PR` only if your output improves the shared surface for others

## How To Work With Practicals

Use a `fork` when:

- you are exploring
- you are learning privately
- you are still messy
- the output is personal, incomplete, or experimental

Open a `PR` when:

- you improved docs
- you cleaned structure
- you added a reusable checklist or guide
- you made a practice task clearer
- you added something that helps the next learner

Good public outputs:

- a glossary
- a beginner guide
- a validation checklist
- a comparison table
- a resource ladder
- a dashboard mockup
- a workflow map

## Practical Reading Rule For MVPs

`MVP` is not its own sphere.

It is a delivery stage.

That means:

- a science-facing MVP may be `S+E` or `S+E+T`
- a software startup MVP is often `E+T`
- a research-informed product can be `S+E+T`
- a training or coaching MVP may live in `Bridge` or `E`

Ask one question:

`What is the main question behind this build right now?`

- If it is about understanding a real-world domain, start from `S`
- If it is about validating value or demand, start from `E`
- If it is about tooling or implementation, start from `T`
