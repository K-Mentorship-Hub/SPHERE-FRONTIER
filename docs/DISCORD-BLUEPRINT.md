# Discord Blueprint

## Recommendation

Do not start with `S / E / T` multiplied by every function if activity is still low.

That creates nine channels before the community is even alive.

## Best Lean Launch

### Core channels

- `#start-here`
- `#announcements`
- `#academy`
- `#mvp-lab`
- `#networking`
- `#resources`
- `#wins-and-blockers`

### Optional private channels

- `#founder-room`
- `#k-spam`
- `#admin-ops`

## How To Represent Spheres Without Duplicating Channels

Use:

- roles
- post prefixes
- thread naming
- forum tags

Examples:

- `[S]`
- `[E]`
- `[T]`
- `[S+T]`
- `[E+T]`
- `[S+E+T]`

## If You Really Want Sphere Splits Later

Do it only after traffic is real.

Then split into:

- `#academy-s`, `#academy-e`, `#academy-t`
- `#mvp-s`, `#mvp-e`, `#mvp-t`
- `#networking-s`, `#networking-e`, `#networking-t`

## Hall of Fame

Recommended early format:

- one pinned message or markdown page
- monthly highlights
- best contributor
- best helper
- best academy-practice contribution

Do not build a full gamification system yet.
