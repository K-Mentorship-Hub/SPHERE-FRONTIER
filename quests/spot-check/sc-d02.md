# 🔍 Spot Check · Code & Engineering

**Сфера:** T · **Напрям:** 2

## 🎯 Мета
Код «працює на моєму ноуті» — але логіка ламає крайній випадок.

## 📋 Сценарій
```python
users = []
for i in range(1, len(users)):
    print(users[i]['name'])
```
Ментор каже: «Цикл ок для списку користувачів».

## 🕵️ Твоє завдання
1. Що станеться, якщо `users` порожній або з 1 елементом?
2. Чому `range(1, len(users))` пропускає першого?
3. Як переписати безпечніше для «першого практичного»?

## ⚠️ Пастка
Працює на happy path ≠ коректна логіка.

---

<details>
<summary>✅ Перевір себе (лише після своєї відповіді)</summary>

Порожній список — цикл не виконається (ок). Один елемент — теж пропуск (баг, якщо треба всіх). `range(1, len)` **завжди** пропускає index 0.

**Fix:** `for user in users:` або `for i in range(len(users))` з перевіркою порожнього списку.

</details>
